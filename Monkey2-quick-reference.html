Classes

  • Class identifier [ < generic-types > ] [ Extends type ] [ Implements types ] [ Abstract | Virtual | Final ] [ Where where-expression ] [ = extern-symbol ]
         class-members
    End | End Class

Class methods are final by default and must be declared Virtual or Abstract if they are designed to be overridden, or Override or Override Final if they override a superclass method.

This behavior can be modified by declaring a class Virtual. Classes declared Virtual or with a superclass declared Virtual behave much like monkey1 or java classes - all methods are virtual by default, and will silently override any existing superclass virtual methods. It is an error to declare methods of such classes either Virtual or Override.

Classes declared Abstract or that contain any abstract methods cannot be instantiated using New.

Classes declared Final cannot be extended.

Example:

Struct S

    Private

    'Internal storage for the property
    Field _seconds:Float

    Public

    'Read/Write 'Seconds' property...
    Property Seconds:Float()

        Return _seconds

    Setter( seconds:Float )

        _seconds=seconds
    End

    'Read only 'Minutes' property...
    Property Minutes:Float()

        Return _seconds/60.0
    End

End
